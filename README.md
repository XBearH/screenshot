# 轻量级桌面截图工具 PRD（产品需求文档）
版本：v1.1（新增自动翻译需求）
最后更新：2026-03-07

---

## 1. 文档概述
### 1.1 文档目的
本文档定义**轻量级桌面截图工具**的产品需求、功能边界与非功能约束，用于指导开发、测试与产品验收，确保产品核心价值落地。
### 1.2 文档范围
覆盖 v1.1 版本核心功能，包含：系统级截图捕获、自动保存、状态反馈、**截图自动翻译**；不包含复杂标注、云同步等扩展能力（后续版本迭代）。
### 1.3 受众
- 产品/开发/测试团队
- 项目协作方
- 开源社区贡献者

---

## 2. 产品定位与核心目标
### 2.1 产品定位
一款**无干扰、跨平台、高效**的系统级桌面截图工具，区别于市面带复杂标注/编辑功能的截图软件，主打「一键触发、自动保存、智能翻译」的极简体验，满足用户快速捕获并理解屏幕内容的核心需求。
### 2.2 核心目标
1.  实现**系统级**全屏/指定窗口截图能力，精准捕捉桌面内容
2.  全程**零用户交互**（无手动框选、无标注步骤），一键完成截图
3.  跨平台兼容 Windows / macOS / Linux 三大桌面系统
4.  高性能处理：4K 分辨率截图秒级生成，低资源占用
5.  自动保存截图至本地，支持基础路径配置
6.  **新增**：截图完成后自动识别文字并翻译，快速获取外文内容

## 3. 用户角色与使用场景
### 3.1 用户角色
| 角色 | 核心需求 |
|------|----------|
| 普通办公用户 | 快速截取会议/文档内容，无需复杂操作，自动保存方便后续查找 |
| 开发者/设计师 | 截取调试界面/设计稿，用于版本对比、问题反馈 |
| 技术支持人员 | 截取系统错误界面，快速提交工单/分享给同事 |
| **外语学习者/跨境工作者** | **截取外文界面/文档，自动识别并翻译，快速理解内容** |

### 3.2 典型使用场景
1.  **场景1：快速截取全屏**  
    开发者遇到界面 bug，点击「一键截取全屏」按钮，截图自动保存到桌面，无需额外操作即可粘贴到工单。
2.  **场景2：截取指定窗口**  
    设计师需要截取设计工具窗口，选择目标窗口后一键保存，避免截取无关桌面内容。
3.  **场景3：后台触发截图**  
    用户通过快捷键触发截图，无需打开应用主界面，后台完成捕获与保存。
4.  **场景4：外文内容自动翻译（新增）**  
    用户浏览外文技术文档，截取包含关键内容的区域，系统自动识别文字并翻译成中文，展示翻译结果。

| 多显示器支持 | 识别多屏幕环境，允许选择目标屏幕 | 1. 自动枚举所有屏幕源<br>2. 提供简单选择方式（默认主屏幕，可配置） | P1 |

| 功能点 | 需求描述 | 交互逻辑 | 优先级 |

| 路径配置 | 允许用户自定义保存路径（v1.0 极简版） | 1. 提供基础设置入口，选择本地文件夹<br>2. 配置持久化，重启后生效 | P1 |

## 5. 非功能需求
### 5.1 性能需求
- 4K 分辨率截图处理时间 ≤ 2 秒
- **OCR+翻译总耗时 ≤ 5 秒**（针对常见文字密度场景）
- 应用内存占用 ≤ 500MB（空闲状态），截图+翻译过程中峰值 ≤ 1GB
- 连续截图+翻译 100 次无崩溃、无明显内存泄漏
### 5.2 兼容性需求
- 操作系统：Windows 10+、macOS 12+、Linux（Ubuntu 20.04+ 主流发行版）
- 分辨率：支持 1080P/2K/4K 等常见桌面分辨率
- 多显示器：兼容 1-4 屏幕布局
- **OCR/翻译服务**：支持离线/在线两种模式，离线模式优先保证可用性
### 5.3 安全与隐私需求
- 不收集任何用户数据、截图内容，所有处理仅在本地完成（离线模式）
- 在线模式下，仅将截图文字内容上传至翻译服务，不传输原始图片
- 首次使用需引导用户授权「屏幕捕获」权限，明确告知权限用途
- 不主动上传任何数据至云端，无后台网络请求（在线翻译模式除外）
### 5.4 可用性需求
- 核心功能（全屏截图+自动翻译）学习成本 ≤ 1 分钟，新手可直接使用
- 界面简洁无冗余元素，核心操作按钮突出
- 错误提示清晰易懂，避免技术术语，提供解决方案引导
- 翻译结果展示清晰，方便用户快速查阅与复制
### 5.5 稳定性需求
- 应用启动成功率 ≥ 99.9%
- 截图功能成功率 ≥ 99%（排除用户未授权等主动拒绝场景）
- **OCR+翻译成功率 ≥ 95%**（排除无文字/模糊文字场景）
- 长时间后台运行无卡顿、无崩溃

## 6. 技术选型（面向开发侧）
### 6.1 核心技术栈
- **前端框架**：React + TypeScript + Vite（高效开发、类型安全）
- **桌面 runtime**：跨平台桌面应用框架（提供系统级 API、进程管理、文件系统访问能力）
- **核心技术**：
  - 系统桌面捕获 API：获取屏幕/窗口媒体流
  - Canvas API：图像处理、格式转换
  - IPC 通信：实现渲染进程与主进程的安全交互
  - **OCR 技术**：Tesseract.js（离线）/ 百度智能云 OCR（在线）
  - **翻译技术**：translate.js（离线）/ 谷歌翻译 API / 腾讯云翻译 API（在线）
- **构建工具**：Vite + 框架配套打包工具（生成各平台安装包）
### 6.2 架构约束
- 采用「主进程-渲染进程」分离架构，保障安全与性能
- 核心截图、OCR、翻译逻辑封装为独立模块，便于后续扩展与维护
- 配置数据持久化至本地文件，避免依赖第三方存储
- **OCR/翻译模块**：支持插拔式替换，方便切换离线/在线方案

## 7. 界面与交互设计（极简版）
### 7.1 主界面布局
- 顶部：应用标题 + 版本信息
- 中间：核心操作区（「一键截取全屏」按钮，占主要视觉权重）
- 底部：
  - 左侧：状态反馈区 + 最新截图预览
  - 右侧：**翻译结果面板**（折叠/展开，默认展开）
### 7.2 交互流程（新增翻译后）
1.  用户打开应用 → 展示主界面
2.  点击「一键截取全屏」→ 按钮变为禁用状态，显示 loading
3.  后台完成截图捕获 → 自动保存至桌面
4.  自动启动 OCR → 识别截图文字
5.  自动启动翻译 → 生成译文
6.  弹出成功提示，展示文件路径，底部更新截图预览与翻译结果
7.  用户可复制译文或保存翻译结果
### 7.3 视觉风格
- 深色模式为主（护眼，适合长时间使用）
- 简洁扁平化设计，无冗余装饰
- 操作按钮高对比度，突出核心功能
- **翻译结果面板**：分栏展示原文/译文，使用不同颜色区分，方便阅读

## 9. 风险与应对
| 风险点 | 影响 | 应对方案 |
|--------|------|----------|
| 系统授权拦截 | 首次使用无法截图 | 界面明确引导用户授权，授权失败时提供清晰提示与解决步骤 |
| 跨平台兼容性差异 | 部分系统截图/翻译失败 | 封装兼容层，分平台测试，针对不同系统适配捕获/识别 API |
| 大分辨率性能卡顿 | 用户体验下降 | 优化 Canvas 绘制逻辑，分块处理图像数据，降低内存占用 |
| 快捷键冲突 | 与系统/其他应用快捷键冲突 | 提供默认快捷键，允许用户自定义，避免占用系统常用组合键 |
| **OCR 识别准确率低** | **翻译结果偏差大** | **1. 优先针对清晰文字场景优化<br>2. 模糊/手写文字场景提示识别可能不准确<br>3. 支持用户手动修正识别文字** |
| **翻译服务依赖网络** | **离线场景无法使用翻译** | **1. 提供离线翻译模式（精度稍低）<br>2. 无网络时自动切换离线模式，提示用户** |
| **翻译 API 限额/费用** | **大规模使用受限** | **1. 开源版默认使用免费/离线方案<br>2. 提供付费 API 配置入口，供高级用户选择** |
